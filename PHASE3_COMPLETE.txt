================================================================================
                          TECHNICAL DETAILS
================================================================================

Model Architecture:
───────────────────
Input: (batch, 10, 1) - 10 normalized numbers

CNN Extraction Stage:
  Conv1D(64, kernel=3)    → (batch, 10, 64)
  BatchNormalization      → (batch, 10, 64)
  Conv1D(32, kernel=3)    → (batch, 10, 32)
  BatchNormalization      → (batch, 10, 32)
  MaxPooling1D(2)         → (batch, 5, 32) ← 50% smaller!
  Dropout(0.2)            → (batch, 5, 32)

LSTM Processing Stage:
  LSTM(64, ret_seq=True)  → (batch, 5, 64)
  Dropout(0.2)            → (batch, 5, 64)
  BatchNormalization      → (batch, 5, 64)
  LSTM(32, ret_seq=False) → (batch, 32)
  Dropout(0.2)            → (batch, 32)
  BatchNormalization      → (batch, 32)

Prediction Stage:
  Dense(32)               → (batch, 32)
  Dropout(0.3)            → (batch, 32)
  Dense(16)               → (batch, 16)
  Dropout(0.2)            → (batch, 16)
  Dense(10 + softmax)     → (batch, 10)

Output: (batch, 10) - probabilities for numbers 0-9

Total Parameters: 95,074 (vs 150K before)

Why It's Faster:
────────────────
1. MaxPooling(2) reduces 10 → 5 timesteps (50% smaller for LSTM)
2. Smaller LSTM layers (64→32 vs old 128→64)
3. Fewer parameters overall (95K vs 150K)
4. All optimized for CPU inference
5. Conv1D is very efficient on CPU

Why It's More Accurate:
──────────────────────
1. CNN learns LOCAL PATTERNS (structure in 3-number windows)
2. LSTM learns TEMPORAL PATTERNS (how structures evolve)
3. Together they see more context than LSTM alone
4. Feature hierarchy learned automatically
5. Better generalization with less overfitting

Why It's CPU-Optimized:
──────────────────────
1. MaxPooling drastically reduces computation
2. Fewer total parameters (less memory)
3. Conv1D layers are lightweight
4. No unnecessary layers
5. Efficient for both training and inference

================================================================================
                          ROLLBACK PROCEDURE
================================================================================

If anything goes wrong (unlikely), rollback is simple:

1. Stop system:
   docker-compose down

2. Revert code:
   git checkout trainer/train.py

3. Restart:
   docker-compose up -d --build

Takes ~30 seconds total. System reverts to old LSTM.

Note: You can backup new model before rollback:
   cp models/lstm_model.keras models/lstm_model.keras.phase3
